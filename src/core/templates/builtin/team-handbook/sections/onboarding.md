# Onboarding Guide

This guide will help you get set up and productive in your first few weeks.

## Your First Day

### Morning

- [ ] Meet with {{TEAM_LEAD}} for welcome and overview
- [ ] IT setup: laptop, accounts, badges
- [ ] Office tour and introductions
- [ ] Slack and email setup

### Afternoon

- [ ] Review team handbook (this document!)
- [ ] Set up development environment
- [ ] Meet your onboarding buddy
- [ ] Join {{SLACK_CHANNEL}} and introduce yourself

## Your First Week

### Week 1 Goals

- [ ] Complete all required HR paperwork
- [ ] Set up all development tools and access
- [ ] Read through team documentation
- [ ] Attend all regular team meetings
- [ ] Make your first code contribution
- [ ] Meet with all team members 1-on-1

### Daily Check-ins

You'll have daily 15-minute check-ins with {{TEAM_LEAD}} to:

- Review progress
- Address any blockers
- Answer questions
- Set goals for the day

## Your First Month

### Month 1 Goals

- [ ] Ship your first feature or bug fix
- [ ] Present in a team meeting
- [ ] Complete onboarding training modules
- [ ] Shadow customer support for a day
- [ ] Review and understand the codebase architecture
- [ ] Participate in code reviews

### Weekly 1-on-1s

After week 1, you'll have weekly 1-on-1s with {{TEAM_LEAD}} to discuss:

- Progress and accomplishments
- Challenges and concerns
- Learning and development goals
- Feedback and questions

## Onboarding Checklist

### Access and Accounts

- [ ] GitHub account and team access
- [ ] Jira/Project management tool
- [ ] Slack workspace
- [ ] Google Workspace (email, calendar, drive)
- [ ] VPN access
- [ ] AWS/Cloud console access
- [ ] Monitoring and logging tools
- [ ] Design tools (Figma, etc.)

### Development Setup

- [ ] Clone repositories
- [ ] Install development dependencies
- [ ] Set up local development environment
- [ ] Run tests successfully
- [ ] Start local development server
- [ ] Configure IDE with team settings

### Documentation

- [ ] Read team handbook
- [ ] Review architecture documentation
- [ ] Understand deployment process
- [ ] Learn incident response procedures
- [ ] Study coding standards and guidelines

### Training

- [ ] Complete security training
- [ ] Complete compliance training
- [ ] Attend architecture overview session
- [ ] Shadow customer support
- [ ] Attend sprint planning and retro

### Social

- [ ] Meet all team members
- [ ] Join team lunch or coffee
- [ ] Attend team social event
- [ ] Connect on professional networks

## Onboarding Buddy Program

You'll be paired with an onboarding buddy who will:

- Answer day-to-day questions
- Help you navigate the organization
- Introduce you to people and processes
- Provide context and background
- Be your go-to person for anything

Your buddy is NOT your manager - feel free to ask them anything!

## Common Questions

### How do I get help?

Ask in {{SLACK_CHANNEL}}! The team is friendly and responsive.

### How do I request access to tools?

Create a ticket in the IT support system or ask your manager.

### Where do I find X?

Check the team wiki or ask in Slack. We're working on making things easier to find!

### What should I work on?

Your manager will assign initial tasks. Look for issues tagged "good first issue" in Jira.

### How often do we deploy?

We deploy to production multiple times per day through our CI/CD pipeline.

## Timeline Expectations

- **Day 1:** Environment setup and access
- **Week 1:** First code contribution
- **Week 2:** First pull request merged
- **Week 4:** Ship first feature
- **Month 2:** Full productivity
- **Month 3:** Onboarding complete

Remember: Everyone learns at their own pace. These are guidelines, not strict deadlines.
