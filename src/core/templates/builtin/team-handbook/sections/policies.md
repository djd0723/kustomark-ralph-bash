## Time Off

### Vacation / PTO

**Policy:**
- Unlimited PTO (within reason)
- Minimum 2 weeks recommended per year
- Encourage taking time off

**Process:**
1. Discuss with {{TEAM_LEAD}} in advance
2. Submit request in HR system
3. Update team calendar
4. Set Slack auto-responder
5. Document handoff

**Guidelines:**
- Give 2+ weeks notice for >1 week off
- Avoid critical sprint milestones
- Coordinate with team
- Actually disconnect when off!

### Sick Days

**Policy:** Take the time you need to recover

**Process:**
- Notify {{TEAM_LEAD}} and team in {{SLACK_CHANNEL}}
- No documentation needed for <3 days
- Mark calendar as OOO
- Hand off urgent work

**Message Template:**
```
Not feeling well today, taking a sick day.
Will be back tomorrow/[date].
[Contact person] can help with urgent items.
```

### Personal Days

**Use For:**
- Medical appointments
- Family matters
- Personal errands
- Mental health

**Just let the team know:**
```
Taking a few hours this afternoon for an appointment.
Available on Slack if urgent.
```

### Parental Leave

**Policy:**
- 16 weeks primary caregiver
- 8 weeks secondary caregiver
- Flexible return options

**Contact HR for details**

## Security and Confidentiality

### Data Classification

**Public:**
- Marketing materials
- Published blog posts
- Open source code

**Internal:**
- Team communications
- Internal documentation
- Development code

**Confidential:**
- Customer data
- Financial information
- Unreleased features
- Employee information

**Restricted:**
- Security credentials
- Legal matters
- M&A information
- Executive communications

### Security Best Practices

**Credentials:**
- Use 1Password for all passwords
- Enable MFA everywhere
- Rotate keys quarterly
- Never commit secrets

**Devices:**
- Encrypt hard drives
- Lock screens when away
- Keep software updated
- Report lost devices immediately

**Data:**
- Don't share customer data externally
- Use secure channels for sensitive info
- Delete local data when no longer needed
- Report security incidents immediately

**Working in Public:**
- Don't discuss confidential topics
- Lock screen when away
- Use privacy screens
- Be aware of surroundings

### Incident Reporting

**If You Suspect a Security Issue:**
1. Stop what you're doing
2. Report to security@{{COMPANY_NAME}}.com
3. Don't investigate further
4. Preserve evidence
5. Don't discuss publicly
